javascript Promise

        javascript promise are Easy to manage when dealing with multiple asynchronous operation where callbacks can create hell leading to unmanageable
        code. prior to promise events and callback function were used but they had limited functionality and created unmanageable code. Mutiple 
        callback functions would create callback hell that leads to unmanageable code. promises are used to handle asynchronous operation in javascript

        let promise=new promise(function(resolve,reject)
        {
            //do something
        });

        parameters:
        1)The promise constructor takes only one argument which is a callback function
        2)the callback function takes two argument,resolve and reject
        @perform operation inside the callback,function and if everything went well then call resolve.
        @if desire operations do not go well then call reject  

        A promise has four states:

        1)Fulfilled:Action realted to the promise succeeded
        2)rejected:Action related to the promise succeeded
        3)pending:Promise is still pending i.e not fultilled or rejected yet
        4)settled:promise has been fulfilled or rejected 

        Promise Consumers:Promise can be consumed by registering functions using.then and .catch method 

        1)promise then()method: It is invoked when a promise is either resolved or rejcted.it may also be defined as a carrier 
        that takes data from promise and further executes it successfully

        Parameters:It takes two function as parameters.
        @The first function is executed  if the promise is resolved and a result is received.
        @The second function is executed if the promise is rejected and an error is received.It is optional and there is better way to handle
        error using .catch() method 

        2)promise catch()method It invoked when a promise is either rejected or some error has occurred in execution It is used as an 
        error handler whenever at any step there is a chance of getting an error

        Parametersit takes one function as parameter
        @function to handle errors or promise rejections catch()method internally calls then (null,errorhandler)i.e .catch is just shorthand for .then(errorhandler)
